ACP-77 is not a simple static curve. Its fee depends on how long the network stays above the target. So for the ACP-77 surface I expose that assumption directly. Below the target it stays at the minimum. Above the target it compounds according to the public ACP-77 / Build.avax description.

Current ACP-255: strongest original thesis, weakest against the aggregate-fee objection
Monotonic: cleanest safety story, biggest redesign
Hybrid: best compromise if the goal is to preserve the 0→10k growth story and still clean up the post-threshold economics

How to read the explorer

  • 3D surfaces: X = total network validators V, Z = L1 validator count n, Y = monthly L1 fee.
  • First 2D chart: compare all four formulas across n while holding V fixed.
  • Second 2D chart: compare all four formulas across V while holding n fixed.
  • Hybrid model: uses the current ACP-255 formula up to 9k validators, blends through 9k→11k, then switches to a fitted monotonic curve.

The hybrid is the version I find most defensible today. Not because it is mathematically prettier than the pure monotonic curve, but because it preserves the original intuition of ACP-255 where it matters most, then gets stricter exactly where reviewers are right to worry.

ACP-77

feeRate = M * exp(x / K), with x = max(x + V - T, 0). For the explorer I convert that into a monthly fee under a sustained-over-target assumption.

ACP-255, current Gaussian version

total = n * M * multiplier(n) * networkFactor(V). This is the proposal exactly as drafted in the ACP review cycle.

ACP-255, pure monotonic replacement

total = 50 + 35 * log2(n). This is the cleanest response to the resource-pricing objection, but also the sharpest departure from the current ACP-255 story.

ACP-255, hybrid transition model

current below 9k, blend between 9k and 11k, monotonic fitted branch above 11k. This is the version that seems to work best.
